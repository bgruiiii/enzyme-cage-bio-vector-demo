# 微生物侧特征升维与统一 256 维表征简要计划

## 1. 核心问题

当前 Bio Vector R3 已将反应、酶、底物和微生物四类对象统一映射到 256 维向量空间。但四类模态进入模型前的信息量并不平衡：

| 模态 | 当前输入 | 输入维度 | 输出维度 |
|---|---|---:|---:|
| 反应 | DRFP 反应指纹 | 2048 | 256 |
| 底物 | Morgan fingerprint | 2048 | 256 |
| 酶 | ESM-C 表征 | 1152 | 256 |
| 微生物 | 人工汇总代谢特征 | 28 | 256 |

因此，反应、底物和酶是“高维信息压缩到 256 维”，而微生物侧更像是“低维摘要被扩展到 256 维”。当前微生物 28 维只包含核心连通性汇总、16 个辅因子/货币分子化学计量信息，以及主反应物/主产物覆盖情况，未显式使用具体代谢物层面的偏好信息。

## 2. 可利用的新信息来源

2026-05-25 已完成 RHEA core-preference full HPC run。该结果基于 2,867 个 CarveMe 微生物代谢模型和 NIS Core 结果，判断 RHEA 反应中的每个 BigG target 代谢物是否能够接入对应微生物的核心代谢网络。

该 long CSV 的每一行代表：

```text
微生物模型 × RHEA 反应 × target BigG 代谢物
```

主要字段包括 target 代谢物 ID、反应物/产物角色、是否在原始模型中、是否在核心网络中、是否可达核心网络等。它比当前 28 维摘要保留了更细粒度的代谢物身份和网络连通性信息。

## 3. 统计结果

已完成本地审计，关键结果如下：

| 项目 | 数量 |
|---|---:|
| 当前训练样本 | 145,607 |
| 当前微生物输入 | 28 维 |
| long target-level 行数 | 692,151 |
| long unique target BigG metabolite IDs | 1,154 |
| stoichiometry BigG metabolite IDs | 1,122 |
| BigG-only V2 合并候选维度 | 13,780 |
| 去除低信息代谢物后候选维度 | 约 13,540 |

矩阵审计还显示，未归一化高维特征会被水、质子、ATP 等高频通用代谢物主导；列 L2 归一化后，候选矩阵的 top-48 effective rank 从 4.20 提升到 46.25，说明高维信息确实存在，但必须进行归一化和低信息特征控制。

## 4. 建议方案

首版 V2 建议采用 **BigG-only 可解释特征**，暂不把 placeholder 代谢物作为主训练输入。核心设计如下：

| 模块 | 预计维度 | 含义 |
|---|---:|---|
| 当前 28 维基础特征 | 28 | 保留 R3 已使用的汇总信息 |
| BigG signed stoichiometry | 1,122 | 每个 BigG 代谢物的净化学计量系数 |
| BigG role-split stoichiometry | 2,244 | 分开记录反应物侧和产物侧 |
| Long target preference channels | 10,386 | target 代谢物 × 连通状态/角色/模型存在性 |

推荐流程：

```text
约 13.5k-13.8k 维 BigG 稀疏特征
→ 列归一化 / 分块缩放 / 低信息代谢物降权
→ 投影或 SVD/PCA 到中间维度
→ 微生物 encoder 输出 256 维
→ 与现有四模态统一空间兼容
```

不建议只把当前 28 维用更大的 MLP 扩展到高维，因为这不会增加真实输入信息。升维应来自具体代谢物身份和核心网络连通性信息的显式展开。

## 5. 需要确认的关键点

1. 首版是否采用 BigG-only 作为主方案，placeholder 仅作为后续消融或补充分析。
2. 水、质子、ATP、ADP、NADH、CO2、Pi 等低信息通用代谢物是过滤、降权，还是保留但归一化。
3. V2 首要目标是改进 row-level 微生物-反应匹配，还是同步构建 assembly-level 微生物整体代谢画像。
4. 降维路线优先采用端到端 projector，还是先用 SVD/PCA 降到 512 或 1024 维再进入 projector。

## 6. 结论

微生物侧当前瓶颈不是最终 256 维输出，而是输入信息量过低。已有 long CSV 提供了大量尚未被充分利用的 target-level 代谢网络信息。基于 BigG 代谢物身份、反应角色和核心网络连通性，可以把微生物输入从 28 维提升到约 13.5k 维可解释稀疏特征，再统一降到 256 维。该方案能更接近反应、底物和酶的“高维信息压缩”逻辑，并保持与现有 R3 统一向量空间兼容。
